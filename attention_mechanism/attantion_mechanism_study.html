When you read the sentence "I like Natural Language Processing", your brain doesn't process each word in isolation. To understand what you like, you automatically connect "I" → "like" → "Processing" (the whole field). Your attention weighs how relevant every other word is to the current word you're interpreting.

Transformers replicate this mechanism mathematically. For each token, it asks: "How much should I attend to every other token when computing my representation?"

Parallelizable Context-aware Long-range dependencies Scales to billions of tokens

Before attention, RNNs processed sequences step-by-step, compressing all context into a fixed-size hidden state. This caused:

  • Vanishing gradients for long sequences
  • Information bottleneck — earlier tokens get "forgotten"
  • Sequential computation — can't be parallelized on GPUs

In "I like Natural Language Processing", an RNN reaching the final token might have already diluted information about "I" — the subject of the sentence.

Attention solves this by allowing every token to directly attend to every other token — no bottleneck. Each token gets a weighted mix of all others' information, where the weights are learned dynamically based on content.

Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V Where Q = Query, K = Key, V = Value, dₖ = key dimension

Tokenization — Breaking the Sentence

Our sentence is first split into tokens. Each token will be embedded into a vector of dimension d_model. For this walkthrough we use d_model = 4 for clarity (real models use 512–4096).

Q, K, V — The Three Projections

Each token's embedding x is linearly projected into three vectors using learned weight matrices Wᴼ, Wᴷ, Wᵛ:

Think of it like a database lookup: the Query is your search term, the Keys are the database index labels, and the Values are the actual data stored. You compute similarity of your Query against all Keys to determine how much of each Value to retrieve.

3
Attention Scores — Q · Kᵀ

For each token acting as a Query, we compute a dot product with every token's Key. This gives a raw score representing compatibility. We then scale by √dₖ to prevent vanishingly small gradients.

score(i, j) = Qᵢ · Kⱼ / √dₖ e.g. score("like", "NLP") = Q_like · K_NLP / √4

Softmax — Normalizing to Probabilities

The raw scores are passed through softmax, converting them into a probability distribution (values sum to 1.0). Higher scores → higher attention weights.

αᵢⱼ = exp(scoreᵢⱼ) / Σₖ exp(scoreᵢₖ) Attention weights for token i attending to all tokens j

These weights represent how much each token attends to every other token. The heatmap below shows the full 5×5 attention matrix for our sentence:

5
Weighted Value Sum — The Output

Finally, each token's output is a weighted sum of all Value vectors, where the weights are the softmax attention scores we just computed:

outputᵢ = Σⱼ αᵢⱼ · Vⱼ Each output is a blend of all Values, weighted by attention

For example, the output for "like" will be a mix of all Value vectors, but heavily weighted toward "Processing" (high attention) and less toward "Natural" — because "like" semantically connects to the thing being liked.

6
Multi-Head Attention

Real transformers run multiple attention heads in parallel. Each head learns to focus on different types of relationships — syntax, semantics, coreference, etc. Their outputs are concatenated and projected.

MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · Wᴼ Where headᵢ = Attention(QWᵢᴼ, KWᵢᴷ, VWᵢᵛ)
Head 1

Syntactic relations
"like → I (subject)"
